# 音乐下载插件

为 EchoMusic 增加当前曲目的「音频 / 歌词 / 封面」分别下载能力，并提供独立下载弹窗、播放栏按钮注入和全局快捷键。

## 功能

- **三种资源分开下载**，互不影响：
    - **音频文件**：递归扫描当前曲目对象中的所有 `http(s)` 字段，按音质优先级自动选出最高音质直链。
    - **歌词**：从播放快照 `snapshot.lyric.lines` 取内嵌歌词，导出为带时间轴的 `.lrc`。
    - **封面图片**：按字段名（`cover | pic | img | image | album`）自动探测封面直链。
- **独立下载弹窗**：每个资源一行，含音质/类型标签、**可编辑的下载地址输入框**和下载按钮。
    - 自动探测并填入最佳地址，用户可手动修改 / 粘贴后再下载。
    - 当前没有播放曲目时，也可手动粘贴地址下载。
- **多入口触发**：命令、快捷键、设置面板按钮、播放栏注入按钮（注入失败回退为右下角浮动按钮）。
- 下载走浏览器原生下载，保存到浏览器**默认下载目录**；跨域 `fetch` 受限时自动降级为新标签打开链接。

## 打开方式

| 入口                       | 说明                                          |
|--------------------------|---------------------------------------------|
| 命令 `music-download.open` | 通过 `ctx.commands.register` 注册，任何能触发命令的地方都可用 |
| 快捷键 `⌘/Ctrl + Shift + D` | 通过 `ctx.shortcuts.register` 注册，全局可用         |
| 设置面板「打开下载面板」按钮           | 位于设置 → 音乐下载                                 |
| 播放栏下载按钮                  | 注入到播放器底栏操作区；找不到稳定容器时回退为右下角浮动按钮              |

## 下载弹窗

弹窗是一个 DOM 浮层（`#echo-dl-panel`），不是宿主浮窗。每行资源包含：

- **标签与音质/类型标记**：音频显示探测到的音质（如 `FLAC / 无损`），歌词显示行数，封面显示来源字段。
- **可编辑地址输入框**：默认填入自动探测到的地址，可手动修改或粘贴新地址。
- **下载按钮**：点击后按当前输入框地址下载。

下载流程：优先 `fetch`（`mode: cors`、`credentials: omit`）拿到 `blob` 并以自定义文件名保存；失败（跨域 / 混合内容 / HTTP
错误）时降级为 `<a download>` 原生下载（新标签打开）。

## 音质探测优先级

音频地址从当前曲目对象中递归收集所有 `http(s)` 字段，再用路径关键词排序取最高优先级：

| 优先级 | 关键词（路径中匹配，不区分大小写）                     | 显示标签         |
|:---:|---------------------------------------|--------------|
|  6  | `hires` `hi-res` `master` `dsd` `dsf` | Hi-Res       |
|  5  | `flac` `lossless` `ape` `wav`         | FLAC / 无损    |
|  4  | `sq` `320` `exhigh`                   | SQ · 320kbps |
|  3  | `hq` `192`                            | HQ · 192kbps |
|  2  | `128` `standard` `low`                | 标准 · 128kbps |
|  1  | 其他可识别音频地址                             | 标准           |

> 若以上都匹配不到，会尝试读取曲目的 `quality / level / rate / bitrate / q` 字段作为音质标签，仍无则显示「标准」。
> 不同音源的曲目字段名差异较大，如果自动探测的音频地址不对，请把控制台 `[music-download] 下载探测` 日志反馈给开发者。

## 文件命名

下载文件名统一为 `歌手 - 标题.扩展名`：

- 歌手 / 标题取自当前曲目或播放快照（`title | name | songName`、`artist | singer`）。
- 文件名中的 `\\ / : * ? " < > |` 会替换为 `_`；为空时回退为 `download`。
- 扩展名优先从 URL 中提取；音频兜底 `mp3`，封面兜底 `jpg`，歌词固定为 `lrc`。

## 歌词导出

- 来源：`snapshot.lyric.lines`，逐行取 `text | content | words`。
- 时间戳格式 `[mm:ss.xx]`；时间单位按秒处理，个别以毫秒给出的来源（>1000）会自动换算。
- 没有时间信息的行作为纯文本保留。
- 导出 MIME 为 `text/plain;charset=utf-8`。

## 设置

设置位于「设置 → 音乐下载」，存储 key 为 `music-download:settings:v1`。

| 设置项          | 说明                    |
|--------------|-----------------------|
| 启用音乐下载插件     | 插件总开关                 |
| 在播放器底栏注入下载按钮 | 是否尝试把下载按钮注入到播放器底栏     |
| 注入失败显示浮动按钮   | 注入失败时是否在右下角显示浮动按钮作为回退 |

## 控制台调试日志

每次打开下载面板，插件都会在控制台打印 `[music-download] 下载探测` 分组，包含：

- `currentTrack` 完整对象
- `snapshot` 播放快照
- 所有探测到的 URL 候选字段（`path = url`）
- 最终选中的音频 / 封面字段与歌词行数

> 自动探测不准时，请把这组日志反馈给开发者，以便校准字段映射。

## 限制与说明

- 下载文件保存到浏览器**默认下载目录**，不支持自定义路径（受浏览器安全策略限制）。
- 跨域 / 混合内容资源无法 `fetch` 为 `blob`，会自动降级为新标签打开链接，此时文件名可能由服务器决定。
- 全程对宿主 API 采用可选链防御式调用，任一 API 缺失都不会让插件崩溃。

## 关于窗口声明（island 浮窗）

`manifest.json` 的 `contributes.windows` 声明了一个 `island` 浮窗，`island.js` / `island.css` 是对应的历史遗留入口。

> **当前下载功能并未使用该浮窗。** 下载面板是运行在主窗口里的 DOM 浮层（`#echo-dl-panel`），主入口 `index.js` 也没有调用
`ctx.windows.show("island")`。该声明与文件仅为兼容保留，可在确认不再需要后安全移除。

如未来需要把下载面板迁移为独立浮窗，可参考仓库根目录 `docs/windows.md`（插件浮窗与 Now Playing API）使用 `ctx.windows` /
`ctx.nowPlaying`。

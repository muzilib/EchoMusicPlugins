# 音乐下载插件

为 EchoMusic 增加当前曲目的「音频 / 歌词 / 封面」分别下载能力，并提供独立下载弹窗。

## 功能

- 三种资源**分开下载**，互不影响：
    - 音频文件：自动按 `Hi-Res → FLAC/无损 → SQ → HQ → 标准` 优先级探测最高音质直链。
    - 歌词：从播放快照取内嵌歌词，导出为带时间轴的 `.lrc`。
    - 封面图片：自动探测封面直链。
- **独立弹窗**展示，每个资源都有一个**可编辑的下载地址输入框**：
    - 自动探测填入最佳地址，用户可手动修改 / 粘贴后再下载。
    - 当前没有播放曲目时，也可手动粘贴地址下载。
- 下载走浏览器原生下载，保存到浏览器**默认下载目录**；跨域 `fetch` 受限时自动降级为打开链接。
- 入口（满足不同场景）：
    - 命令 `music-download.open` 与快捷键 `Ctrl/⌘ + Shift + D`（一定可用）。
    - 设置面板里的「打开下载面板」按钮。
    - 尝试把下载按钮注入到**播放器底栏**；注入失败则回退为右下角**浮动按钮**。

## 打开弹窗时的控制台日志

每次打开下载面板，插件都会在控制台打印 `[music-download] 下载探测` 分组，包含：

- `currentTrack` 完整对象
- `snapshot` 播放快照
- 所有探测到的 URL 候选字段（`path = url`）
- 最终选中的音频 / 封面字段与歌词行数

> 不同音源的曲目字段名差异较大。如果自动探测的音频地址不对，请把这组日志反馈给开发者，以便校准字段映射。

## 设置

- 启用音乐下载插件
- 在播放器底栏注入下载按钮
- 注入失败显示浮动按钮

## 文件说明

| 文件                         | 说明                                            |
|----------------------------|-----------------------------------------------|
| `index.js`                 | 下载插件主入口（命令 / 快捷键 / 设置面板 / 弹窗 / 播放栏按钮注入，含内联样式） |
| `manifest.json`            | 插件清单                                          |
| `island.js` / `island.css` | 历史遗留的灵动岛浮窗入口（当前下载功能未使用，保留未删）                  |

## 要求

- EchoMusic `>=2.2.6`
